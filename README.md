# Система онлайн-консультаций и вопросов-ответов

## Описание проекта

Веб-приложение для онлайн-консультаций и обмена вопросами-ответами между пользователями и экспертами. Проект разработан в рамках курсового проекта по дисциплине ПМ04 "Проектирование и обеспечение бесперебойной работы web-сайта".

## Функциональность

### Основные возможности:
- ✅ Регистрация и аутентификация пользователей
- ✅ Создание и просмотр вопросов
- ✅ Добавление ответов на вопросы
- ✅ Система категорий вопросов
- ✅ Запись на консультации
- ✅ Личный кабинет пользователя
- ✅ Адаптивный дизайн для всех устройств
- ✅ Валидация форм на клиентской и серверной стороне
- ✅ Безопасное хранение паролей (bcrypt)
- ✅ JWT аутентификация

## Установка и запуск

### Предварительные требования:
- Node.js (v16 или выше)
- MongoDB (v5 или выше)
- npm или yarn

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd consultation-qa-system
```

### 2. Установка зависимостей

#### Backend:
```bash
cd server
npm install
```

#### Frontend:
```bash
cd client
npm install
```

### 3. Настройка переменных окружения

Создайте файл `.env` в папке `server/` со следующим содержимым:

```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/consultation-qa
JWT_SECRET=your_secret_key_here
NODE_ENV=development
```

### 4. Запуск MongoDB

Убедитесь, что MongoDB запущена:

```bash
# Windows
net start MongoDB

# Linux/Mac
sudo systemctl start mongod
```

### 5. Запуск приложения

#### Запуск Backend (в папке server/):
```bash
npm run dev
```
Сервер запустится на http://localhost:5000

#### Запуск Frontend (в папке client/):
```bash
npm run dev
```
Приложение откроется на http://localhost:3000

## API Endpoints

### Аутентификация
- `POST /api/auth/register` - Регистрация пользователя
- `POST /api/auth/login` - Вход в систему

### Вопросы
- `GET /api/questions` - Получить все вопросы
- `GET /api/questions/:id` - Получить вопрос по ID
- `POST /api/questions` - Создать вопрос (требуется авторизация)
- `POST /api/questions/:id/answers` - Добавить ответ (требуется авторизация)
- `GET /api/questions/my` - Получить мои вопросы (требуется авторизация)

### Консультации
- `GET /api/consultations` - Получить все консультации
- `GET /api/consultations/:id` - Получить консультацию по ID
- `POST /api/consultations` - Создать консультацию (требуется авторизация)
- `PUT /api/consultations/:id` - Обновить консультацию (требуется авторизация)
- `GET /api/consultations/my` - Получить мои консультации (требуется авторизация)

## Резервное копирование

### Резервное копирование проекта:

**Windows:**
```bash
backup.bat
```

**Linux/Mac:**
```bash
chmod +x backup.sh
./backup.sh
```

### Резервное копирование базы данных:

```bash
chmod +x backup-db.sh
./backup-db.sh
```

Резервные копии сохраняются в папках `backups/` и `db_backups/`
